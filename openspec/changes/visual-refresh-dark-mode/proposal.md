## Why

GachaPlanner 当前视觉设计存在三个核心问题：颜色系统落地不一致（9 个页面、17 个组件使用硬编码颜色而非 CSS 变量）、关键文本对比度不足（warning 色在浅色背景上对比度仅 1.6:1，主色文本对比度 2.8:1，均低于 WCAG AA 标准）、以及完全缺失深夜模式。随着用户对深色模式的需求日益普遍（iOS/Android 系统级深色模式已普及），小程序的视觉体验需要一次系统级的美术刷新，统一颜色规范、修复对比度缺陷、并引入自动跟随系统的深色模式。

## What Changes

- **统一颜色系统**：将所有硬编码颜色替换为 CSS 变量，建立唯一颜色来源（`app.wxss` 变量系统）
- **修复对比度缺陷**：调整主色、warning 色、muted 文本色的色值，确保在浅色背景下满足 WCAG AA 标准（≥4.5:1）
- **引入深色模式**：建立完整的深色配色系统（Dark Mode），包含背景、卡片、文本、边框、阴影的反转映射
- **自动跟随系统**：支持跟随 iOS/Android 系统深色模式开关自动切换（`wx.onThemeChange`）
- **手动切换入口**：在「我的」页面增加深色/浅色/跟随系统三种模式的手动切换
- **导航栏适配**：导航栏背景色和文本色随模式动态调整（使用自定义导航栏实现）
- **图表组件适配**：概率图表、曲线图表在深色模式下使用反色配色
- **阴影系统重构**：深色模式下将黑色阴影替换为亮色发光或更高不透明度阴影

## Capabilities

### New Capabilities

- `dark-mode-system`: 完整的深色模式配色系统与动态切换能力，包含浅色/深色两套设计令牌、自动跟随系统主题、手动切换入口。
- `color-system-unification`: 颜色系统统一化，建立唯一颜色来源，消除硬编码颜色，所有页面和组件使用 CSS 变量。

### Modified Capabilities

- `overview-display`: 总览页的视觉展示需求变更——目标卡片、冲突提示、策略卡片等组件需要支持深色模式下的背景色、文本色、状态色反转。

## Impact

- **全局样式**：`app.wxss` 变量系统全面重构，新增 `--dark-*` 变量系列
- **Pages**: 所有 12 个页面的 wxss 需审查并替换硬编码颜色
- **Components**: 所有 22+ 个组件的 wxss 需审查并替换硬编码颜色
- **Charts**: `probability-chart`、`resource-curve-chart`、`history-trend-chart` 的 canvas 绘制逻辑需增加深色模式配色分支
- **App Config**: `app.json` 导航栏配置需改为自定义导航栏以支持动态背景色
- **Storage**: 新增用户主题偏好存储键（`theme_preference`）
- **No breaking changes**: 旧数据完全兼容，浅色模式为默认，深色模式为新增能力
