## ADDED Requirements

### Requirement: 目标表单保存按钮视觉层级提升
目标表单的保存/更新按钮 SHALL 使用主色调背景、全宽显示，并在视觉上明显区别于次要操作（删除按钮）。

#### Scenario: 玩家设定新目标
- **WHEN** 玩家填写目标表单并准备保存
- **THEN** 「设定目标」按钮使用主色调（#C4A77D）背景、白色文字、全宽圆角按钮样式
- **AND** 按钮高度不小于 88rpx，确保舒适的触控区域

#### Scenario: 玩家编辑已有目标
- **WHEN** 玩家编辑已有目标
- **THEN** 「更新目标」按钮使用主色调背景、白色文字、全宽显示
- **AND** 「删除目标」按钮使用次级样式（浅色背景 + 红色文字），位于保存按钮下方

### Requirement: 移除对概率计算无实质影响的字段
目标表单 SHALL 移除「目标类型」（角色/武器/其他）字段。该字段不影响概率计算模型，且增加表单复杂度。

#### Scenario: 玩家设定目标
- **WHEN** 玩家打开目标表单
- **THEN** 表单只显示：目标名称、期望命座/精炼、大保底开关、卡池开始/结束日期
- **AND** 不显示「目标类型」选择器

### Requirement: 命座/精炼输入增加范围校验
目标表单的「期望命座/精炼」输入 SHALL 限制输入范围为 0-6（角色）或 0-5（武器），超出范围时给出提示。

#### Scenario: 玩家输入有效命座
- **WHEN** 玩家输入 0 到 6 之间的整数
- **THEN** 输入正常接受

#### Scenario: 玩家输入无效命座
- **WHEN** 玩家输入小于 0 或大于 6 的数字
- **THEN** 自动修正为边界值（小于 0 修正为 0，大于 6 修正为 6）
- **AND** 显示 Toast 提示「命座范围为 0-6」

### Requirement: 跳转规划页自动滚动到目标表单
从总览页点击「添加目标」或「编辑目标」跳转到规划页时，页面 SHALL 在加载完成后自动滚动到目标表单区域。

#### Scenario: 从总览页添加目标
- **WHEN** 玩家在 overview 页点击「+ 目标」或「添加目标」
- **AND** 跳转到 planning 页且携带 autoFocusTarget=true 参数
- **THEN** 页面加载完成后，自动 scroll 到目标表单区域
- **AND** 目标表单自动展开

#### Scenario: 从总览页编辑目标
- **WHEN** 玩家在 overview 页点击目标卡片的「编辑」菜单
- **AND** 跳转到 planning 页且携带 autoFocusTarget=true 参数
- **THEN** 页面加载完成后，自动 scroll 到目标表单区域
- **AND** 目标表单自动展开并预填充已有数据
