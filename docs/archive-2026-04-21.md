## 二、功能问题修复

### 问题1: 数据同步问题 ✅

**问题描述**：规划页修改资源后，首页显示不一致

**根本原因**：
- `gameService.currentGameId` 与 `app.globalData.selectedGameId` 不同步
- 首页通知机制依赖页面实例存在

**修复方案**：
- `index.js:270` - 点击游戏卡片时同步更新 `gameService.currentGameId`
- `planning.js:149` - 保存后通知首页刷新

**涉及文件**：
- `pages/index/index.js`
- `pages/planning/planning.js`

---

### 问题2: 总览页面缺失 ✅

**问题描述**：缺少所有游戏的总体达成情况展示

**修复方案**：新增总体规划页面

**页面结构**：
```
TabBar:
├── 首页 (资源总览)
├── 总览 (所有游戏达成情况) ← 新增
├── 历史 (抽卡记录)
└── 我的 (设置)

规划页面 ← 改为普通页面，从首页/总览进入
```

**涉及文件**：
- `pages/overview/overview.js`
- `pages/overview/overview.wxml`
- `pages/overview/overview.wxss`
- `app.json` - TabBar 配置调整

---

### 问题3: 导航流程问题 ✅

**问题描述**：规划页在 TabBar 中，返回逻辑不清晰

**修复方案**：
- 将规划页移出 TabBar
- 使用 `navigateTo` 跳转，支持返回
- 从首页或总览页进入规划页

**涉及文件**：
- `pages/index/index.js` - 使用 `navigateTo`
- `pages/overview/overview.js` - 使用 `navigateTo`
- `app.json` - TabBar 精简为4个

---

### 问题4: 抽数计算错误 ✅

**问题描述**：把原石数量直接当作抽数显示

**修复方案**：
- 正确计算公式：`总抽数 = Math.floor(原石/160) + 纠缠之缘`
- 所有显示统一为抽数，不再显示原石数量

**涉及文件**：
- `pages/index/index.js` - 首页统计逻辑
- `pages/overview/overview.js` - 总览统计逻辑
- `components/game-card/` - 游戏卡片组件
- `components/overview-card/` - 总览卡片组件

---

### 问题5: 保存提示烦人 ✅

**问题描述**：每次修改资源都弹出"已保存"提示

**修复方案**：
- 移除 Toast 提示
- 改为静默保存，只在 console 记录日志

**涉及文件**：
- `pages/planning/planning.js:142` - 删除 `wx.showToast`

---

### 问题6: 错误提示干扰 ✅

**问题描述**：规划页修改数据时弹出"数据加载失败"提示

**根本原因**：
- 规划页通知首页刷新时，首页 loadData 的 catch 块触发 Toast

**修复方案**：
- 首页错误处理改为完全静默
- 添加单个游戏的错误捕获
- 最终 catch 块完全不显示 Toast

**涉及文件**：
- `pages/index/index.js` - loadData 方法重构

## 三、页面结构优化

### 最终页面架构

```
小程序结构：
├── pages/
│   ├── index/              # 首页 - 资源总览
│   ├── overview/           # 总览 - 所有游戏达成情况
│   ├── planning/           # 规划 - 单个游戏详细编辑
│   ├── history/            # 历史 - 抽卡记录
│   ├── profile/            # 我的 - 设置
│   ├── rule-settings/      # 规则设置
│   └── config-manager/     # 配置管理
│
└── components/
    ├── overview-card/      # 总览卡片
    ├── game-card/          # 游戏卡片
    ├── resource-input/     # 资源输入
    ├── probability-display/ # 概率显示
    └── probability-chart/  # 概率图表
```

### 用户操作流程

```
首页 → 查看所有游戏概览
  └─> 点击游戏卡片 → 进入规划页
      └─> 编辑资源、查看概率
      └─> 点击返回 → 回到首页

总览 → 查看所有游戏达成情况
  └─> 点击游戏卡片 → 进入规划页
      └─> 编辑资源、查看概率
      └─> 点击返回 → 回到总览页

我的 → 规则设置 → 自定义概率参数
    → 配置管理 → 导出/导入/分享配置
```

## 四、数据流优化

### 数据同步机制

```
用户修改资源（规划页）
  ↓
gameService.updateResources() 保存到 storage
  ↓
通知首页刷新（indexPage.loadData）
  ↓
首页重新计算并更新显示
  ↓
✅ 数据一致，无提示干扰
```

### 抽数计算逻辑

```javascript
// 正确的抽数计算
const totalPulls = Math.floor(原石 / 160) + 纠缠之缘;

// 首页统计
总抽数 = Σ (各游戏抽数)

// 资源分配
分配比例 = (游戏抽数 / 总抽数) × 100%
```
